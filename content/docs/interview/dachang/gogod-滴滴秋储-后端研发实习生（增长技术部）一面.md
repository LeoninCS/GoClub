### 1. Redis 五种数据结构？讲讲 zset 底层

答：ZSet 常见实现是“字典 + 跳表”。字典按 member 快速定位 score，跳表按 score 维护有序集合，支持范围查询、排名和排行榜。小数据量时 Redis 会使用 listpack 节省内存。

### 2. Redis 为什么快？

答：Redis 主要基于内存操作，数据结构高效，网络模型采用事件驱动，单线程执行命令减少锁竞争，整体延迟很低。

### 3. Go 的 GC 讲讲

答：Go GC 采用并发标记清扫，配合三色标记和写屏障降低停顿。STW 主要出现在标记准备、标记终止等短阶段。

### 4. MySQL 的索引有哪些？

答：InnoDB 主键索引是聚簇索引，叶子节点保存整行；二级索引叶子节点保存主键值，查询其他列需要回表。覆盖索引能直接从索引返回结果。

### 5. MySQL 的锁有哪些？隔离级别有哪些？不同的隔离级别是怎么用这些锁的

答：MySQL 常见锁有行锁、表锁、间隙锁、临键锁和意向锁。隔离级别越高，并发冲突控制越强，InnoDB 通过 MVCC 和锁共同保证一致性。

### 6. MySQL 有哪些日志？有什么用？

答：MySQL 回答要围绕表结构、索引、事务、锁、日志和执行计划展开。场景题优先说明查询路径、并发风险和可观测指标。

### 7. 消息队列为什么用 Kafka？在你项目里对比其他的有什么优势吗

答：Kafka 适合高吞吐、可回放、日志流和大规模消费组场景。选择时要结合吞吐、延迟、消息模型、运维成本和业务语义。

### 8. HTTP 和 HTTPS 有什么区别？

答：HTTPS 是 HTTP 运行在 TLS 之上。TLS 通过证书校验身份，协商会话密钥，后续使用对称加密保护传输内容。

### 9. TCP 为什么要四次挥手？第四次挥手省略之后会有什么问题？

答：三次握手确认双方收发能力和初始序列号。四次挥手源于 TCP 全双工关闭。TIME_WAIT 用于重传最后 ACK，并等待旧报文在网络中消失。
