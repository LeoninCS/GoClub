### 6. varchar 和 char 的区别

答：`char` 是定长，适合长度固定的数据；`varchar` 是变长，适合长度变化大的字段。`char` 可能浪费空间但读取和比较更稳定，`varchar` 更省空间但有额外长度记录。

### 7. 给一个查询语句和联合索引，问会不会走索引

答：InnoDB 主键索引是聚簇索引，叶子节点保存整行；二级索引叶子节点保存主键值，查询其他列需要回表。覆盖索引能直接从索引返回结果。

### 8. 怎么排查慢查询

答：先用慢查询日志定位 SQL，再用 `EXPLAIN` 看 type、key、rows、Extra，确认索引命中、回表、临时表和 filesort。随后结合业务基数优化索引、改写 SQL、减少返回列，必要时做分页、缓存或异步化。

### 13. 为什么延迟队列可以用 zset 来做？

答：ZSet 的 score 可以存任务触发时间戳，member 存任务 ID 或任务内容。消费者按当前时间取 `score <= now` 的任务，抢到后删除或标记执行，适合实现轻量延迟队列。

### 14. zset 的底层数据结构是什么样的？

答：ZSet 常见实现是“字典 + 跳表”。字典按 member 快速定位 score，跳表按 score 维护有序集合，支持范围查询、排名和排行榜。小数据量时 Redis 会使用 listpack 节省内存。

### 15. 输入一个 URL 到网页加载之间发生了什么？

答：浏览器先做 DNS 解析，再建立 TCP/TLS 连接，发送 HTTP 请求，服务端返回资源，浏览器解析 HTML、CSS、JS，构建 DOM/CSSOM 并完成布局绘制。

### 16. TCP 为什么要三次握手，而不是两次？

答：三次握手确认双方收发能力和初始序列号。四次挥手源于 TCP 全双工关闭。TIME_WAIT 用于重传最后 ACK，并等待旧报文在网络中消失。
