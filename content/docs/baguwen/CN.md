# 计算机网络

这里整理计算机网络面试里的常见问题，覆盖 DNS、TCP、HTTP、HTTPS 和排障思路等主题。

## 基础

### 输入一个 URL 会发生什么？

答案：先解析域名，再建连发请求，服务端返回资源，浏览器解析渲染，最后连接复用或关闭。

关键点：
- DNS 解析（本地缓存 -> 递归/权威解析）。
- TCP 三次握手；HTTPS 还包含 TLS 握手。
- 浏览器构建 DOM/CSSOM，布局绘制。

### 域名和 IP 的关系是什么？

答案：域名是便于记忆的名字，IP 是网络地址，DNS 负责把域名映射到 IP。

关键点：
- 一个域名可对应多个 IP（负载均衡）。
- 多个域名也可指向同一 IP（虚拟主机）。

### DNS 是什么协议？解析流程是怎样的？

答案：DNS 是应用层协议，常用 UDP 53，分层分布式解析。

关键点：
- 典型路径：根域名 -> 顶级域 -> 权威域名服务器。
- 大报文或区域传送会用 TCP。
- 结果会按 TTL 缓存。

### 网站访问异常时如何定位问题？

答案：按“应用层 -> 网络层 -> 解析层”逐层排查。

关键点：
- 看状态码（4xx/5xx）。
- `ping`、`traceroute/tracert` 查连通性与链路。
- `nc/telnet` 测端口，`nslookup/dig` 查 DNS。

## HTTP

### HTTP/1.1、HTTP/2、HTTP/3 的区别

答案：核心是传输效率和队头阻塞治理能力不断提升。

关键点：
- HTTP/1.1：文本协议，复用能力弱。
- HTTP/2：二进制分帧、多路复用、HPACK。
- HTTP/3：基于 QUIC（UDP），握手和弱网表现更好。

### HTTPS 的 S 是什么？核心原理是什么？

答案：`S` 是 Secure，表示 HTTP 之上加了 TLS 安全层。

关键点：
- 非对称用于协商密钥。
- 对称用于业务数据加密。
- 证书链用于身份校验，防中间人。

### 为什么有了 HTTP，还要 RPC 和 WebSocket？

答案：三者关注点不同，分别适配“通用接口、服务间高效调用、实时双向通信”。

关键点：
- HTTP：跨平台通用。
- RPC：内部服务性能与治理更好。
- WebSocket：长连接、服务端主动推送。

## TCP

### TCP 为什么是三次握手？为什么不是两次？

答案：三次握手用于双方确认收发能力和初始序列号，确保连接状态可靠。

关键点：
- 客户端和服务端都要确认“我发你能收、你发我能收”。
- 两次握手无法充分确认客户端接收能力。

### 三次握手和四次挥手分别是什么？

答案：握手是 `SYN -> SYN+ACK -> ACK`，挥手是 `FIN -> ACK -> FIN -> ACK`。

关键点：
- 握手建立连接。
- 挥手释放连接。
- 挥手常是四次因为全双工双方独立关闭。

### TCP 和 UDP 的核心区别

答案：TCP 可靠有序但开销高；UDP 轻量低延迟但不保证可靠。

关键点：
- TCP：连接、重传、流控、拥塞控制。
- UDP：无连接、尽力而为。
- 场景：TCP 偏事务传输，UDP 偏实时音视频/游戏。

### TCP 的滑动窗口、流量控制、拥塞控制是什么？

答案：三者分别解决“发送效率、接收方承载能力、网络承载能力”。

关键点：
- 滑动窗口：提高链路利用率。
- 流量控制：防止接收端被打爆。
- 拥塞控制：慢启动、拥塞避免、快重传、快恢复。
